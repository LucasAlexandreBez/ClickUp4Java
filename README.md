## 🕵️‍♂️ Why should I use?

All projects have a lot of tracking lists, tasks, dashboards and many other software development and quality tracking.
But all of then will have the same problem:. **C O M M U N I C A T I O N**

When we are developing or testing we always forgot to change the status of a task, add comments or even we need to create a new task as we didn't noticed that would be requried for the story or feature.

And if you are the PO/PM that is reallyyy annoying as you need to have all this small details updated then on any status call you are aware of what is happening.

Another things is the comments that die on silence inside the repository. Well, not anymore as we will keep track with the annotations and codes that we call, so when you are with some doubts of what this test case or code was implementing you can check the task ID and track in ClickUp to get more detais  as now the **tracking live togheter with the code**.

## 📦 Instalation

- Add at you maven/gradle:
### Maven
 ```xml
    <dependency>
        <groupId>com.github.lucasalexandrebez</groupId>
        <artifactId>clickup4java</artifactId>
        <version>LAST_RELEASE</version>
    </dependency>
```
### Gradle
```gradle
implementation "com.github.lucasalexandrebez:clickup4java:LAST_RELEASE"
```
- Or download the Jar files and implement manually
